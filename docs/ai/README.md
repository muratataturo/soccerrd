
## サッカーNote の AI 設計方針

1. **入力多様性**: 試合動画・テキスト所見・スコアシートなど、選手 / コーチが「思考と言語化」を行うあらゆる入力を許容する
2. **構造化された蓄積**: 出力は単なる文章ではなく、後段で動的検索・横断検索可能な構造（タグ・概念・時系列・選手ID）を持つ
3. **還元設計**: ノートが追加されるたびに、過去の関連ノート・組織知が自動で関連付け / 要約される
4. **eval 駆動**: モデル変更・プロンプト変更は **必ず eval を通す**。評価セットは [`evals/`](./evals/) に蓄積する

## AI を扱う際のルール

- プロンプトは AI 機能ファイル (`features/<name>.md`) に必ず添付する（口頭プロンプトは残さない）
- モデル / プロンプト変更時は eval スコアの before/after を `features/<name>.md` の更新履歴に記録する
- ハルシネーション・誤示唆は **P0 リスク** として扱う（ユース・コーチの判断材料になるため）
- プライバシー（選手の個人データ）取り扱いは ADR（[`engineering/decisions/`](../engineering/decisions/)）に必ず判断を残す
